Quickstart

Get started with Human+ in under 5 minutes.

1. Get Your API Key

Sign up at /human-plus/dashboard to get your API key.

2. Make Your First Request

# Using curl
curl -X POST https://api.usecami.com/api/v1/humanplus/guide \
  -H "Authorization: Bearer hp_your_api_key" \
  -H "Content-Type: application/json" \
  -d '{"text": "I am so frustrated with my coworker"}'

3. Use the Guidance

The API returns actionable guidance you can inject into your LLM's system prompt:

{
  "guidance": {
    "emotional_state": "raw_emotional_release",
    "approach": "validate_only",
    "tone": "warm, validating",
    "flags": ["witness_feeling", "avoid_problem_solving"],
    "depth": "moderate",
    "ethics_clear": true
  },
  "system_injection": "The user is in a raw emotional release state. They need witnessing, not solutions. Validate the feeling first. Tone: warm, validating. Be present with the feeling rather than managing it. Don't jump to solutions.",
  "processing_ms": 45
}

Authentication

All API requests require authentication via Bearer token.

Authorization: Bearer hp_your_api_key

Security: Keep your API key secret. Don't expose it in client-side code. Always make API calls from your server.

Guide Endpoint (Standard Tier)

Returns actionable guidance without exposing raw metrics. IP-protected.

POST /api/v1/humanplus/guide

Analyze text and return emotional intelligence guidance.

Request Body

ParameterTypeRequiredDescription
textstringYesThe user input to analyze
contextstringNoDomain context (e.g., "customer_support", "clinical")

Response

FieldTypeDescription
guidance.emotional_statestringDetected emotional state
guidance.approachstringRecommended approach (validate_only, match_energy, etc.)
guidance.tonestringRecommended tone descriptors
guidance.flagsarrayActionable flags (witness_feeling, avoid_problem_solving, etc.)
guidance.depthstringConversation depth (surface, moderate, deep, profound)
guidance.ethics_clearbooleanWhether the input passes ethical gates
system_injectionstringReady-to-use text for your LLM's system prompt
processing_msintegerProcessing time in milliseconds

Integration Guide

How to integrate Human+ with your existing LLM pipeline.

Python Example

# Python integration example
import requests
import openai

HUMANPLUS_KEY = "hp_your_api_key"
OPENAI_KEY = "sk-your_openai_key"

def get_humanplus_guidance(user_input):
    response = requests.post(
        "https://api.usecami.com/api/v1/humanplus/guide",
        headers={"Authorization": f"Bearer {HUMANPLUS_KEY}"},
        json={"text": user_input}
    )
    return response.json()

def chat_with_alignment(user_input):
    # Step 1: Get Human+ guidance
    guidance = get_humanplus_guidance(user_input)
    
    # Step 2: Check ethics gate
    if not guidance["guidance"]["ethics_clear"]:
        return "I can't help with that request."
    
    # Step 3: Inject guidance into system prompt
    system_prompt = f"""You are a helpful assistant.
    
{guidance['system_injection']}"""
    
    # Step 4: Call your LLM
    response = openai.ChatCompletion.create(
        model="gpt-4",
        messages=[
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": user_input}
        ]
    )
    return response.choices[0].message.content

TypeScript Example

// TypeScript integration
async function chatWithHumanPlus(userInput: string): Promise<string> {
  // Get Human+ guidance
  const guidance = await fetch('https://api.usecami.com/api/v1/humanplus/guide', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${HUMANPLUS_KEY}`,
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({ text: userInput }),
  }).then(r => r.json());

  // Ethics check
  if (!guidance.guidance.ethics_clear) {
    return "I can't help with that request.";
  }

  // Call your LLM with injected guidance
  const response = await openai.chat.completions.create({
    model: 'gpt-4',
    messages: [
      { role: 'system', content: guidance.system_injection },
      { role: 'user', content: userInput },
    ],
  });
  
  return response.choices[0].message.content;
}
